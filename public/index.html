Architecture

완전한 백엔드 구조

Worker API와 CloudPress 자체 상태 엔진 기반 CloudPressDB, 인증/관리자 권한, 프로젝트 프로비저닝, 감사 로그, 보안 헤더를 포함합니다. Durable Objects 없이 자체 제어면 구조로 구성했습니다.

  1. Edge API Gateway: TLS, CSP, HSTS, 요청 ID, rate-limit
  2. Auth Service: PBKDF2 비밀번호 해시, JWT 세션, 관리자 RBAC
  3. CloudPressDB Native Engine: 자체 NoSQL API, 사용자/프로젝트/감사 컬렉션
  4. Provisioner: WordPress, CP3, DB, Static, PHP 사이트 분리 생성
  5. Observability: 감사 로그와 상태 엔드포인트

보안 안정성

강한 보안 헤더, 입력 검증, 세션 만료, salted PBKDF2, 권한 분리, 감사 로그를 기본값으로 적용합니다.

고가용성

상태 비저장 API와 독립 데이터 계층, 헬스 체크, 장애 격리 설계로 99.99% 이상 가용성을 목표화합니다.

경험 중심 UI

상품 생성, 로그인/가입, 관리자 모드, 프로젝트 목록을 한 페이지 콘솔에서 즉시 체험합니다.
